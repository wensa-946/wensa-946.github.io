目录大纲

请你谈谈对xxx的理解 ? ​

  • 含义: xxx是什么
  • 动机: 为什么会诞生xxx, 为了解决xxx问题
  • 作用: 有xxx好处/优点, 能处理什么问题
  • 场景: xxx适合在什么场景下使用

请你手写xxx api ? ​

  • 梳理目标 api 的执行原理和特性
  • 依据执行原理和特性作为思路
  • 使用能实现思路的基础api编写代码

面试回答如何更突出 ​

  • 第一层：概念 + 定义，八股文
  • 第二层：加上自己的理解和思考，在学习中对它的思考
  • 第三层：在平时工作中的应用，遇到的问题，解决的方案
  • 第四层：通过面试官问你的八股文，你刻意去引导到自己熟悉的领域
    • 考察闭包 => react 流行 hooks 开发 => hooks 使用闭包内存泄漏
    • 考察es6新特性 => Proxy、Reflect => Vue3响应式原理
    • ...

避坑，面试过程中要避免一问一答，面试过程其实是一场技术交流

如何提升硬实力（程序员30岁之前要去培养） ​

自己要养成学习的习惯，每天进步一点点。

学习方法

  • 官方文档
  • 看经典的书 如《java编程思想》
  • github 开源高 star 项目（你感兴趣的）源码不建议去看，不如搞懂原理
  • 优秀的视频教程

学习 => 做项目 => 看大牛如何实现的、他的编程思维是怎么样的? => 自己做自己的项目（模仿 => 超越） => 自己开源

如何培养软实力（程序员30岁之后要去培养） ​

  • 影响力：开源、写技术文章，去帮助更多人，向外输出才能得到外界更多的反馈
  • 人脉：多分享、与志同道合的朋友交流

程序员 8 - 10 年的职业目标、方向 ​

10 年做好一件事情完全足够

  • 进入大厂，达到阿里P8的水平。
  • 在稳定企业深耕，做到管理层。（技术总监/CTO）
  • 技术深度和广度达到一定地步，找到志同道合的朋友创业。
  • 学好英语，打开国外通道，国外没有年龄歧视。
  • 成为开源社区贡献者。
  • 成为技术讲师/技术博主。
